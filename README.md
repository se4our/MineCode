## Тематика сайта, название

Веб-сайт-справочник по использованию командных блоков и команд в игре Minecraft.

__Название проекта: MineCode__

__Суть__: Образовательная платформа, объединяющая теоретические материалы, практические тесты и интерактивный справочник по синтаксису и применению командных блоков и консольных команд Minecraft. Сайт помогает игрокам изучать механику команд, создавать сложные конструкции на серверах и в одиночной игре, создавать карты.

## Цель и задачи системы

__Цель__: Разработать веб-ориентированное учебное пособие для углублённого освоения командных блоков и команд Minecraft, обеспечивающее структурированную подачу теории, проверку знаний и взаимодействие с сообществом.

__Задачи системы:__

1) Реализовать регистрацию и аутентификацию пользователей с разграничением доступа (гость, зарегистрированный пользователь, автор/администратор).
2) Создать иерархическую систему учебных материалов: общая теория командных блоков, раздел синтаксиса команд, расширенные конструкции, особенности работы с модами.
3) Обеспечить интерактивные практические тесты: генерация заданий, ввод команды пользователем, мгновенная проверка корректности синтаксиса и логики.
4) Организовать раздел FAQ с часто задаваемыми вопросами по работе с командами.
5) Создать страницу с информацией об авторах проекта.
6) Предусмотреть административную панель для управления контентом (теория, вопросы тестов).
7) Реализовать адаптивный дизайн, ориентированный на геймерскую аудиторию (тёмная тема, элементы игровой стилистики).

## Целевая аудитория

* **Игроки Minecraft**, желающие освоить или углубить знание командных блоков и консольных команд (от новичков до опытных строителей механизмов).
* **Администраторы серверов Minecraft**, использующие командные блоки для автоматизации, создания мини-игр и управления игровым процессом.
* **Создатели карт, датапаков и модификаций**, которым требуется глубокая справка по синтаксису и возможностям команд.
* **Любители моддинга**, экспериментирующие с модами, расширяющими командный интерфейс (WorldEdit, Custom NPCs, ComputerCraft и др.).

### Отличия от обобщённого пользователя Интернета и их использование в веб-дизайне

- Аудитория преимущественно молодёжная и технически подкованная, привыкшая к гайдам в формате Wiki и интерактивному обучению.
- Ожидание высокой интерактивности и наглядности: схемы, подсветка синтаксиса, примеры кода, мгновенная обратная связь в тестах.
- Желание геймификации: прогресс-бары, достижения, рейтинги по результатам тестов.
- Восприятие сложного технического контента как нормы, ценность точности и глубины информации.
- **Веб-дизайн, учитывающий эти особенности:** тёмная тема с цветовыми акцентами, стилизованными под интерфейс Minecraft (пиксельные шрифты, текстуры, иконки из игры), компактная навигация с поиском, мобильная адаптация, визуализация кода командных блоков как в игре.

**Участник 3 (страницы 11–15):**

## Функциональные возможности

* **Регистрация и аутентификация** с ролями (посетитель, пользователь, администратор). Хранение данных пользователей в базе, защита паролей хешированием.
* **Личный кабинет**: просмотр количества пройденных тестов, лучших результатов, последних активностей.
* **Теоретические статьи**: иерархический каталог, полнотекстовый поиск, подсветка синтаксиса команд с помощью JS-библиотек.
* **Раздел FAQ**: динамический фильтр по поисковому запросу, группировка по категориям.
* **Система практических тестов**:
   - Выборка заданий из базы данных (случайный порядок или фиксированные варианты).
   - Поле ввода команды с валидацией на стороне сервера (регулярные выражения, сравнение с эталоном, допустимыми вариантами параметров).
   - Мгновенное отображение результата проверки и накопление баллов за сессию.
   - Ограничение количества попыток на одно задание для предотвращения накрутки.
* **Администрирование контента**:
   - CRUD-операции для теоретических статей (создание, редактирование, удаление).
   - Управление банком тестовых вопросов: добавление/редактирование задания, указание эталонного ответа и возможных вариаций.
   - Загрузка и вставка изображений (схем, скриншотов) в статьи через файловый менеджер.
* **REST API** (для динамической подгрузки данных, например списка статей, проверки теста через AJAX) с передачей JSON – отвечает критерию «REST API и работа с JSON».
* **Защита от CSRF** для всех форм с использованием механизмов Flask-WTF / Django CSRF.
* **Адаптивная вёрстка** с применением Bootstrap и кастомными стилями в тематике Minecraft.

## Технические аспекты реализации

* **Язык программирования:** Python 3.10+.
* **Веб-фреймворк:** Flask с расширениями (Flask-SQLAlchemy, Flask-Migrate, Flask-Login, Flask-WTF, Flask-RESTful) либо Django (с Django ORM, Django Auth, Django REST Framework).
* **База данных:** SQLite для разработки, PostgreSQL для продакшена (на усмотрение). Использование ORM для абстракции.
* **Шаблонизатор:** Jinja2 – для разделения логики и представления, наследования шаблонов, включения повторяющихся блоков.
* **Статические файлы:** CSS, JavaScript, изображения, шрифты – отдельная директория, раздача сервером разработки или настройка веб-сервером в production.
* **Объём кода:** не менее 900 строк Python-кода (суммарно, без учёта шаблонов и статики), распределённых между участниками равномерно.
* **Система контроля версий:** Git, репозиторий на GitHub/GitLab с осмысленными коммитами и ветками под функционал.
* **Документирование:** комментарии в коде, README с инструкцией по развёртыванию, данный документ с требованиями.
